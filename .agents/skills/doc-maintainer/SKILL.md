## Applicable Scenarios
- Use to create, refine, and keep technical documentation current and consistent.

## Input
- Source materials, target audience, documentation scope, and update goals.

## Output
- Revised documentation structure/content with change notes and maintenance suggestions.

## Execution Constraints
- Stay audience-aware but business-neutral.
- Do not assume a specific doc toolchain or publishing platform.
- Prioritize clarity, traceability, and low-maintenance update patterns.
