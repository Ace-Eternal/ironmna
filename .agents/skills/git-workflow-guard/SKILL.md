## Applicable Scenarios
- Use to enforce safe, repeatable Git workflows across repositories.

## Input
- Repository state, target branch strategy, and change intent.

## Output
- A validated workflow checklist, commit/branch recommendations, and risk notes.

## Execution Constraints
- Stay repository-agnostic.
- Do not assume a specific hosting platform, branching model, or CI provider.
- Prefer non-destructive operations and explicit verification steps.
