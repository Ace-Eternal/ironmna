## Applicable Scenarios
- Use to define and apply consistent code quality and style expectations.

## Input
- Language/tooling context, existing conventions, and desired quality bar.

## Output
- A concise standards checklist with actionable guidance for implementation and review.

## Execution Constraints
- Stay technology- and domain-neutral.
- Do not require proprietary linters, frameworks, or internal policies.
- Keep rules clear, testable, and minimally prescriptive.
